ndp.lboard

The Leaderboard class and related objects.

Description

This module is used to create csv files of data for the video game leaderboards example in the package, ndp. In the following documentation, we illustrate this by creating five different csv files. Each file will contain the data for 10 players.

We first construct a Leaderboard instance with 10 players. Each player on the leaderboard is a Player object that contains data about the player as well as a list of historical scores for that player. The number of scores on the list is random and the marginal distribution of each score is the gamer distribution from the ndp.tools module. To construct such a player, we must decide on the mean number of scores on the list, and also the shape and scale parameters of the gamer distribution.

The shape parameters are $r$ and $a$. We will let $a = 3$, so that in our game, the player has 3 failures before the game is over. The parameter $r$ determines the decay rate of skill level in the general population. We want to set $r$ so that this decay rate matches those seen in TETRIS_SCORES.

If we fit the scores in TETRIS_SCORES to a power law distribution, we get the following:

>>> from ndp.lboard import *
>>> from scipy.stats import pareto
>>> pareto.fit(TETRIS_SCORES, floc=0)[0]
2.361453182701752

The resulting exponent is approximately $7/3$. We therefore set $r = 7/3$.

We will set the scale parameter $c$ so that the average score is approximately $50$. Since the mean of the gamer distribution is $cr/(r - 1)$, we will set $c = 28$, which gives a mean of $49$.

Finally, we set the mean number of historical scores for each player to be $2.4\sqrt{10}$, which is about 7.6. To explain this, note that if there are 10 players, and every game generates a unique score, then we will see $24\sqrt{10}$ unique scores. It is then as if we have 10 players rolling a $24\sqrt{10}$-sided die $2.4\sqrt{10}$ times. If we compare this to 320 thumbtacks being flicked (2 outcomes) 9 times, we see that $$ 10 \cdot 24\sqrt{10} \cdot 2.4\sqrt{10} = 320 \cdot 2 \cdot 9. $$ In this way, a 10-player leaderboard ought to generate a model of comparable complexity to Liu's thumbtack example.

The makeBoard function will construct our leaderboard for us. This function randomly regenerates its data until the total number of scores in the leaderboard is between 72 and 80. Before calling this function, we can check how many times it is expected to loop.

>>> from math import sqrt
>>> meanRunTime(2.4 * sqrt(10))
7.895576353625911

Since this number seems reasonable, we will go ahead and create a leaderboard.

>>> lboard = makeBoard((7/3, 3), 28, 2.4 * sqrt(10))

Although this would be enough to get us started, we want to add one more column to this spreadsheet. We want to add a column that gives the player's long-term average score, according to an NDP model. We will do this by updating the estAvg property of each Player object in the leaderboard.

We first build the base measure of our NDP model.

>>> from ndp.tools import gamer
>>> g = gamer(7/3, 3, scale=28)
>>> baseMeas = [g.cdf(n + 0.5) - g.cdf(n - 0.5) for n in range(499)]
>>> baseMeas.append(g.sf(499.5))

We then extract the data from our leaderboard for the NDP model.

>>> players = sorted(lboard.players, key=lboard.playCountRank)
>>> data = [player.scores for player in players]

Finally, we build the NDP model and add 40,000 weighted simulations.

>>> from ndp import ndpModel
>>> model = ndpModel(1, 1, baseMeas, data=data)
>>> model.scale += 42
>>> model.addSims(40000) # took ~3m on a Macbook
>>> model.ess
326.717352110184

We now compute the estimated law for each player.

>>> import numpy as np
>>> def avg(theta):
>>>     return np.average(range(500), weights=theta)
>>> laws = [model.agentLaw(m, avg) for m in range(10)] # took ~10s

Lastly, we add the estimated mean to each Player object and generate the CSV file.

>>> for m in range(10):
>>>     players[m].estAvg = laws[m].mean
>>> makeCSV(lboard, 'gameexpl1')

So that we have several scenarios to choose from, we generate four more CSV files in the same way.

>>> for i in range(2, 6):
>>>     lboard = makeBoard((7/3, 3), 28, 2.4 * sqrt(10))
>>>     players = sorted(lboard.players, key=lboard.playCountRank)
>>>     data = [player.scores for player in players]
>>>     model = ndpModel(1, 1, baseMeas, data=data)
>>>     model.scale += 42
>>>     model.addSims(40000)
>>>     print(model.ess)
>>>     laws = [model.agentLaw(m, avg) for m in range(10)]
>>>     for m in range(10):
>>>         players[m].estAvg = laws[m].mean
>>>     makeCSV(lboard, f'gameexpl{i}')
12.953846545567364

216def meanRunTime(mean):
217    r'''Returns the expected number of trials until the first success,
218    where a trial involves generating a negative binomial random
219    variable $X$ and success occurs when $72 \le X \le 80$.
220
221    **Arguments:**
222
223    * **mean** (`float`): Used to determine the distribution of $X$. If
224        $\mu$ equals `mean`, then $X$ has a negative binomial
225        distribution with $n = 10$ and $p = 1/\mu$.
226
227    **Returns:**
228
229    * `float`: $1/P(72 \le X \le 80)$.
230
231    '''
232    totalPlayCount = nbinom(10, 1/mean)
233    return 1/(totalPlayCount.cdf(80) - totalPlayCount.cdf(71))

235def makeBoard(args, scale, mean):
236    r'''Makes a leaderboard of 10 players for analyzing with the
237    nested Dirichlet process. The names of the players are taken from
238    the constant, `USER_NAMES`. The `args`, `scale`, and `mean`
239    arguments are passed directly to the `Player` constructor for each
240    player on the leaderboard.
241
242    This function will regenerate the data if the total number of games
243    is not between $72$ and $80$, inclusive.
244
245    To explain why, let $T$ be the total number of games played, and let
246    $n$ be the total number of thumbtacks used in the thumbtack example.
247    Assuming every game produces a unique score, we will have $T$ trials
248    with $T$ observed outcomes. In the thumbtack example, we have $9n$
249    trials with $2$ observed outcomes. For these to be of comparable
250    complexity, we want $T^2 = 18n$.
251
252    In the thumbtack example, we used $n = 320$. To put bounds on the
253    possible complexity of this example, we add and subtract $10\%$,
254    requiring that $288 \le n \le 352$. This gives approximately
255    $72 \le T \le 80$.
256
257    **Returns:**
258
259    * `Leaderboard`: The randomly generated leaderboard.
260
261    '''
262    while True:
263        lboard = Leaderboard([
264            Player(name, args, scale, mean)
265            for name in USER_NAMES
266        ])
267        if 72 <= sum(player.playCount for player in lboard.players) <= 80:
268            return lboard

270def makeCSV(lboard, filename='gameexpl'):
271    r'''Makes a csv file filled with data for the leaderboards example.
272    Will not overwrite a previously existing file.
273
274    The following example code will generate 5 scenarios worth of data:
275
276    ```python
277    >>> from ndp.gamedata import makeCSV
278    >>> for i in range(1, 6):
279    ...     makeCSV(f'gameexpl{i}')
280    ```
281
282    **Arguments:**
283
284    * **lboard** (`Leaderboard`): The leaderboard used to generate the
285        csv file.
286    * **filename** (`str`): The filename to use, without extension.
287        Defaults to "gameexpl".
288
289    **Raises:**
290
291    * **FileExistsError:** If the csv file already exist in the current
292        working directory.
293
294    '''
295    if os.path.exists(filename + '.csv'):
296        raise FileExistsError(f'{filename}.csv already exists')
297
298    with open(filename + ".csv", "w", newline="", encoding='utf-8') as file:
299        writer = csv.DictWriter(file, fieldnames=lboard.fieldnames)
300        writer.writeheader()
301        writer.writerows(
302            sorted(lboard.dictify(), key=lambda row: row['by avg'])
303        )

305class Player:
306    '''A player in the hypothetical game.
307
308    '''
309
310    def __init__(self, name, args, scale, mean):
311        r'''Creates a `Player` object with the given `name` attribute.
312
313        The player's scores have a `gamer` distribution whose shape
314        parameters are given by `args` and whose scale parameter is
315        given by `scale`.
316
317        **Arguments:**
318
319        * **args** (`tuple`(`float`)): A tuple of length 2. The first
320            number is the shape parameter of the Pareto distribution
321            used to determine the `trueAvg` property. The second is the
322            shape parameter of the gamma distribution used to determine
323            the `scores` property.
324        * **scale** (`float`): The scale parameter of the Pareto
325            distribution used to determine the `trueAvg` property.
326        * **mean** (`float`): The mean of the `playCount` attribute.
327
328        '''
329        self.name = name
330        '''`str`: The user name of the player.'''
331
332        self.playCount = np.random.geometric(1/mean)
333        '''`int`: The number of times the player played. This attribute
334        is random and has a geometric distribution. The value of this
335        attribute is generated on object creation, and is then
336        permanently fixed.
337        '''
338
339        self.trueAvg = pareto.rvs(args[0], scale=scale)
340        '''`float`: The theoretical long-term average score of the
341        player. Determined at instance creation by randomly sampling
342        from a `scipy.pareto` distribution.
343        '''
344
345        rawScores = gamma.rvs(
346            args[1],
347            scale=self.trueAvg / args[1],
348            size=self.playCount
349        )
350        self.scores = np.array([round(score) for score in rawScores])
351        '''`list`[`int`]: The scores the player earned. Determined at
352        instance creation by randomly sampling from a `scipy.gamma`
353        distribution with mean `trueAvg`. Scores are rounded to the
354        nearest integer.
355        '''
356
357        self.estAvg = self.avgScore
358        '''`float`: An estimate of the player's theoretical long-term
359        average score. Defaults to the mean of their actual scores, but
360        can be replaced by something more sophisticated, such as an
361        estimate created with the `ndp.tools.NDPModel` class.
362        '''
363
364    @property
365    def highScore(self):
366        '''`int`: The player's highest earned score.'''
367        return max(self.scores)
368
369    @property
370    def avgScore(self):
371        '''`int`: The player's average earned score.'''
372        return np.mean(self.scores)
373
374    @property
375    def diff(self):
376        '''`int`: Returns `avgScore` minus `trueAvg`.'''
377        return self.avgScore - self.trueAvg

379class Leaderboard:
380    '''A leaderboard in the hypothetical game.
381
382    '''
383
384    fields = [
385        'player #',
386        'rank',
387        'by avg',
388        'name',
389        'hi score',
390        'avg score',
391        '# games',
392        'true avg',
393        'est avg',
394        'diff'
395    ]
396    '''`list`[`str`]: The non-score field names used in the `dictify`
397    method.
398    '''
399
400    types = {
401        'name': str,
402        'avg score': float,
403        'true avg': float,
404        'est avg': float,
405        'diff': float
406    }
407    '''`dict`{`str`:`type`}: The non-integer types used in the `dictify`
408    method.
409    '''
410
411    def __init__(self, players):
412        '''Creates a `Leaderboard` instance, assigning the given
413        argument to the `players` property.
414
415        '''
416        self.players = players
417        '''`list`[`Player`]: The list of `Player` objects that are on
418        the leaderboard.
419        '''
420
421        self.fieldnames = self.fields + [
422            f'score {gameNum}'
423            for gameNum in range(
424                max(player.playCount for player in players)
425            )
426        ]
427        '''`list`[`str`]: The full list of field names used in the
428        `dictify` method.
429        '''
430
431    def playCountRank(self, player):
432        '''Returns the ranking of the given player, according to how
433        many games they played. The player with the most games has rank
434        0, and the ranks increase from there. Every player has a unique
435        rank, with ties going to the player that appears first in
436        `players`.
437
438        The play count rankings are intended to be used as indices in an
439        ordered list of players.
440
441        **Arguments:**
442
443        * **player** (`Player`): The player whose rank to return.
444
445        **Returns:**
446
447        * `int`: The rank of the player.
448
449        '''
450        playCounts = np.array([player.playCount for player in self.players])
451        countRanks = rankdata(-playCounts, method='ordinal')
452        return countRanks[self.players.index(player)] - 1
453
454    def highScoreRank(self, player):
455        '''Returns the ranking of the given player, according to their
456        high score. The player with the best high score has rank 1, and
457        the ranks increase from there. Competition ranking is used to
458        handle ties.
459
460        **Arguments:**
461
462        * **player** (`Player`): The player whose rank to return.
463
464        **Returns:**
465
466        * `int`: The rank of the player.
467
468        '''
469        highScores = np.array([player.highScore for player in self.players])
470        highScoreRanks = rankdata(-highScores, method='min')
471        return highScoreRanks[self.players.index(player)]
472
473    def avgScoreRank(self, player):
474        '''Returns the ranking of the given player, according to their
475        average score. The player with the best average score has rank
476        1, and the ranks increase from there. Average scores are rounded
477        to the nearest integer before ranks are computed. Competition
478        ranking is used to handle ties.
479
480        **Arguments:**
481
482        * **player** (`Player`): The player whose rank to return.
483
484        **Returns:**
485
486        * `int`: The rank of the player.
487
488        '''
489        avgScores = np.array(
490            [round(player.avgScore) for player in self.players]
491        )
492        avgScoreRanks = rankdata(-avgScores, method='min')
493        return avgScoreRanks[self.players.index(player)]
494
495    def dictify(self):
496        '''Creates and returns a list of dictionaries suitable for
497        writing to a spreadsheet.
498
499        **Returns:**
500
501        * `list`[`dict`]: A list of dictionaries, one for each `Player`
502            object in the `players` attribute.
503
504        '''
505        dictList = []
506        for player in self.players:
507            d = {
508                'player #': self.playCountRank(player),
509                'rank': self.highScoreRank(player),
510                'by avg': self.avgScoreRank(player),
511                'name': player.name,
512                'hi score': player.highScore,
513                'avg score': round(player.avgScore),
514                '# games': player.playCount,
515                'true avg': player.trueAvg,
516                'est avg': player.estAvg,
517                'diff': player.diff
518            }
519            d.update({
520                f'score {gameNum}': score
521                for gameNum, score in enumerate(player.scores)
522            })
